# IziRig Suite Case Study

## Problem

Auto-rigging tools available at the time were often complex, visually overwhelming or limited for users without advanced rigging knowledge.

The project aimed to simplify the rigging workflow inside Autodesk Maya by reducing steps, improving usability and automating repetitive rigging tasks.

## Solution

IziRig was developed as a Maya tool focused on humanoid auto-rigging.

The workflow allowed the user to:

- Select a character
- Correct character orientation
- Place visual markers
- Configure hands and fingers
- Generate a rig automatically

## Technical Focus

The project involved:

- Python scripting
- Autodesk Maya integration
- Maya API usage
- UI development
- Marker-based positioning
- Rig generation logic
- Controller creation
- Workflow automation

## Software Design

The interface design followed a UX process that included:

- Functional requirements
- User experience strategy
- Wireframes
- Low fidelity design
- High fidelity exploration
- Navigation sitemap

## Result

The project reached a functional alpha version focused on humanoid rig generation.

After the academic delivery, the project continued evolving into IziRig Suite, including complementary tools such as IziCurve and IziRename.

## Limitations

The project did not include:

- Full production-ready testing
- Public source code release
- Complete commercial protection
- Full modular creature rigging implementation

## Portfolio Value

IziRig demonstrates applied software development in a real creative production context, combining Python, automation, UX/UI planning, technical documentation and Autodesk Maya pipeline tooling.
